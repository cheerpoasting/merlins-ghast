1.9.1 How to Read This Specification

The advice for how to read this specification is not what I was originally expecting:

"This specification should be read like all other specifications. First, it should be read cover-to-cover, multiple times. Then, it should be read backwards at least once. Then it should be read by picking random sections from the contents list and following all the cross-references."

2.1 Terminology

IDL
According to MDN, IDL stands for "Interface Description Language". It is a way to talk about "object interfaces" in a language-neutral way. In HTML, the IDL attributes are the attributes that you can read or set using Javascript.
According to Wikipedia, IDL can also mean "Interface Definition Language". It is a generic term for a language that lets one program in one language communicate with a program written in an unknown language. They often describe data types. They are often used to bridge different operating systems and computer languages.
Document
This term is used to refer to any use of HTML, from small static documents to large reports with rich multimedia and even interactive applications.

2.1.2 Resources

Supported
When a format can be used without critical aspects being ignored. For example, a PNG is supported when the pixels can be rendered, even if the animation data isn't known to the parser. However, an MPEG-4 is not supported if the compression format isn't decodeable, even if the dimensions of the video can be determined.
Critical Subresources
Any component that a resource has to have in order to be processed correctly, which differs based on the resource in question.

2.1.8 Conformance Classes

Conforming Documents
"Conforming documents" are those that comply with all the conformance criteria for documents. For readability, some of these conformance requirements are phrased as conformance requirements on authors; such requirements are implicitly requirements on documents. For example, if a requirement statys that "authors must not use the foobar element", it would emply that documents are not allowed to contain elements named foobar.
Conformance Checkers
Conformance checkers must verify that a document conforms to the applicable critera. There are some things that can be checked by a machine -- these things must be checked by a conformance checker. Other things require knowledge of author intent, such as making sure the materials inside a blockquote are actually quotations from some source material.

2.1.9 Dependencies

A large number of important terms are defined in other specifications: Infra, Encoding, URL, HTTP, Fetch, Web IDL, ECMAScript (which is Javascript), DOM, Fullscreen, CSS, and many others.

The default network protocol is HTTP, the default style sheet is CSS, the default scripting is Javascript. IF you use HTTP, you must also implement cookies.
